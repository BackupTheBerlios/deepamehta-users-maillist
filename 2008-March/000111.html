Liebe DeepaMehta-Freunde,

aus der Community kam der Impuls mit DeepaMehta beim Google Summer of  
Code 2008 mitzumachen. Diese Idee finde ich gut, und ich möchte diesen  
Weg gerne gehen. Der Summer of Code ist eine gute Gelegenheit, neue  
DeepaMehta Features zu realisieren und sich mit anderen  
Projekten/Plattformen zu vernetzen.

Ganz unter der Vernetzungsthematik hab' ich für den Summer of Code im  
Moment folgende Entwicklungsideen:

=> Einbindung einer Browser-Engine (z.B. Mozilla Gecko).  
Feature-Beschreibung:
	http://developer.berlios.de/feature/? 
func=detailfeature&feature_id=1764&group_id=5174
=> Anbindung von Blogs, in beide Richtungen:
	- in DeepaMehta: "Blog this Topicmap at Blog XY"
	- im Blog: "Open this Topicmap in DeepaMehta"
=> DeepaMehta als (RESTful) Webservice: DeepaMehta Corporate Memory  
Inhalte für die Entwicklung von Mashups verfügbar machen
=> Einbindung von Google-Search über API (die derzeitige Einbindung per  
SOAP wird von Google nicht mehr unterstützt)
=> Einbindung weiterer Google APIs, z.B. Google Maps (Ankopplung an ein  
"Address"-Topic)
=> Einbindung der Amazon API: Browsen von Produkt-Empfehlungen in Form  
von Topicmaps. Kauffunktion.

Die Idee am Summer of Code teilzunehmen kommt von Urs Lerch.
Malte Reißig hat sich bereits als Mentor zur Verfügung gestellt.
Die Idee mit der Blog-Anbindung und der Amazon API kommt von Vincent  
Scheffler.

Eventuell hat jemand von euch weitere Entwicklungsideen, oder möchte  
sonstwie Unterstützung anbieten. Googles Bewerbungsfrist endet am  
Mittwoch, 12. März.

Heise-Nachricht "Google Summer of Code 2008 nimmt Anmeldungen entgegen":
www.heise.de/newsticker/meldung/104444/from/rss09

Googles "Summer of Code" Homepage:
http://code.google.com/soc/
